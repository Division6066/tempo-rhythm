# Markdown Formatting Guide for TEMPO AI

A comprehensive reference for formatting notes, plans, and task breakdowns in TEMPO. Optimised for ADHD-friendly structure: clear visual hierarchy, bite-sized chunks, and actionable layouts.

---

## Headings

Use headings to create scannable structure. Never skip levels.

```
# H1 — Page/document title (use once)
## H2 — Major section
### H3 — Sub-section
#### H4 — Detail point (use sparingly)
```

**ADHD tip:** Headings act as anchors. A reader should understand the full document from headings alone.

---

## Lists

### Unordered lists

Use for any collection with no fixed order.

```
- Buy oat milk
- Email project proposal
- Schedule dentist
```

### Ordered lists

Use for steps, priorities, or ranked items.

```
1. Open laptop
2. Review today's tasks
3. Pick the single most important thing
```

### Nested lists

Indent with 2 spaces per level. Maximum 3 levels deep.

```
- Work
  - Project Alpha
    - Write spec
    - Schedule review
  - Reply to Slack messages
- Personal
  - Gym at 6pm
```

---

## Task Checklists

Use `- [ ]` for open items, `- [x]` for completed items.

```
- [ ] Draft email to client
- [x] Review yesterday's notes
- [ ] Prepare slides
- [ ] 15-min walk break
```

**ADHD tip:** Keep checklists to 5 items or fewer per section. Long lists cause overwhelm — break them into time blocks.

## Tables

Use tables for comparing options, tracking habits, or scheduling.

```
| Task | Time | Energy | Done |
|------|------|--------|------|
| Email triage | 9:00 AM | Low | [ ] |
| Deep work: spec | 10:00 AM | High | [ ] |
| Lunch break | 12:30 PM | — | [ ] |
| Admin tasks | 2:00 PM | Low | [ ] |
```

**ADHD tip:** Time-block tables are powerful for visual daily planning. See the Daily Plan template below.

## Code Blocks

### Inline code

Use backticks for short snippets, commands, or values.

```
Run `npm install` before starting.
Set `CONVEX_URL` in your `.env` file.
```

### Fenced code blocks

Use triple backticks with an optional language tag.

````
```python
def focus_session(minutes=25):
    print(f"Starting {minutes}-minute focus block")
```
````

### Blockquote

Use `>` to highlight key quotes, observations, or reflections.


### ADHD callout patterns

```
> **Focus:** One task only for the next 25 minutes.

> **Reminder:** Take a 5-minute break after each Pomodoro.

> **Brain dump:** Write everything, judge nothing. Organise after.
```

## ADHD-Friendly Structure Patterns

### Top-3 Daily Priorities

Start every day by picking the three most important tasks. No more.

```
## Today's Top 3

1. **[Most important task]** — Why it matters: [one sentence]
2. **[Second task]** — Deadline: [date/time]
3. **[Third task]** — Estimated time: [X min]
```

### Brain Dump → Structured Format

Step 1: Write everything without judging.

```
## Brain Dump

- need to email Sarah about the budget
- doctor appointment??
- the report is due Friday not Thursday
- buy headphones - current ones broken
- forgot to respond to Alex
- want to read more books
- should update CV
```

Step 2: Sort into categories.

```
## Processed

### Urgent (today)
- [ ] Email Sarah — budget update
- [ ] Reply to Alex

### This week
- [ ] Doctor appointment — call to book
- [ ] Report due Friday — start Wednesday

### Someday
- [ ] Buy headphones
- [ ] Read more (pick one book)
- [ ] Update CV
```

## Summary: ADHD Formatting Principles

1. **Chunk it** — Never more than 5–7 items before a visual break
2. **Prioritise visually** — Use bold, headings, and numbered lists for the most important items
3. **Time-box everything** — Add estimated minutes to every task
4. **Separate capture from processing** — Brain dump first, organise second
5. **One screen, one focus** — Structure so only one section is visible at a time
6. **Use whitespace** — Empty lines and `---` dividers reduce visual overwhelm
7. **Explicit done states** — Use `- [ ]` checklists so completion is obvious
